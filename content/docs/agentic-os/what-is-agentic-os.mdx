Agentic OS turns an AI coding tool into a repeatable work system.

It gives Claude Code and Codex a shared workspace with instructions, skills,
memory, client folders, projects, scheduled jobs, and Command Centre.

## Who it is for

Agentic OS is for people who want their AI tools to remember how they work.

It is useful when you:

* reuse the same methods across many tasks;
* work with more than one client or brand;
* need local files, memory, and outputs to stay organized;
* want a local UI for running work and checking status;
* want a hosted team server later, without changing the local workflow.

## What it gives you

| Area | What it does |
| --- | --- |
| Instructions | Keeps shared operating rules in `AGENTS.md` and Claude-specific wiring in `CLAUDE.md`. |
| Skills | Adds repeatable methods for common work, such as writing, research, diagrams, and operations. |
| Memory | Stores useful context so future sessions can search past decisions and facts. |
| Clients | Separates brand context, memory, projects, jobs, and local settings by client. |
| Command Centre | Gives you a local browser UI for work, files, skills, memory, and team status. |
| Team OS | Lets a hosted server control shared team access, memory, clients, and permissions. |

## Local first

Agentic OS starts as a local workspace on your machine. You can use it alone
without Team OS.

When you add Team OS, the local workspace becomes a client of a hosted server.
The server then decides which team, clients, files, skills, and memory each user
can access.
