Use this section for owner and admin tasks.

Owners and admins can manage Team OS from Command Centre or from the interactive
terminal menu. Use these first when you want a guided path.

## Admin path

| Task | Page |
| --- | --- |
| Start with the UI or terminal menu | [Admin guided path](/docs/admin/admin-guided-path) |
| Choose the right admin surface | [Admin surfaces](/docs/admin/admin-surfaces) |
| Create the first team and owner | [First team and owner](/docs/admin/first-team-owner) |
| Invite and join members | [Members](/docs/admin/members) |
| Manage client grants | [Clients](/docs/admin/clients) |
| Manage skill grants | [Skill grants](/docs/admin/skill-grants) |
| Reset an owner password | [Owner password reset](/docs/admin/owner-password-reset) |
| Check health, memory, and sync | [Health checks](/docs/admin/health-checks) |

Most direct admin commands run from `command-centre/`. Hosted maintenance
commands run inside the `memory-api` container, or from `command-centre/` with
hosted environment variables set.

When a command uses `npm run <script> -- ...`, keep the `--`. It tells npm to
pass the rest of the command to the script.
