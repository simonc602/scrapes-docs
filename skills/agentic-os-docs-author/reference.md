# Agentic OS Docs Author Reference

## Setup page pattern

1. State the outcome.
2. List requirements.
3. Show the command or UI path.
4. Explain the expected result.
5. Link to troubleshooting.

## Concept page pattern

1. Define the concept.
2. Explain why it matters.
3. Give a small example.
4. Link to related tasks.

## Troubleshooting pattern

Use a table when there are several common failures.

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Login fails | Wrong server URL or expired session | Check the URL, sign in again, and run the identity check. |

## Permission writing pattern

When documenting Team OS permissions, name the actor, action, resource, and
result.

Example:

```text
Members can read clients only when an owner or admin grants access.
```
